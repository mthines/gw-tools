# PR Helper Examples

This directory contains examples of how to use the PR Helper skill to generate pull request descriptions.

## Examples

### [auto-clean-feature.md](./auto-clean-feature.md)
**Scenario**: Feature addition with code, tests, and documentation changes
**Type**: `feat`
**Complexity**: High (multiple files, comprehensive changes)

Shows how to generate a PR description for a significant feature addition that includes:
- New functionality
- Updated commands
- Test coverage
- Documentation updates

### [docs-update.md](./docs-update.md)
**Scenario**: Documentation-only update
**Type**: `docs`
**Complexity**: Low (single file, simple change)

Shows how to generate a PR description for a simple documentation change where:
- Only markdown files are modified
- No code or test changes
- Checklist items are appropriately marked N/A

## Using These Examples

1. **Identify your change type**: Compare your changes to the examples above
2. **Follow the similar pattern**: Use the example that matches your scenario
3. **Adapt the template**: Modify the description to match your specific changes
4. **Verify completeness**: Ensure all sections are filled appropriately

## Additional Scenarios

Other common scenarios not yet documented:

- **Bug Fix**: Fixing a specific issue or error
- **Refactor**: Code restructuring without behavior change
- **Test**: Adding or improving test coverage
- **Chore**: Dependency updates, tooling changes

Feel free to add more examples as they occur!
